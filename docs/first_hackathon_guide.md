# First Hackathon Guide

## Mindset

This is not a test of whether you are the smartest person in the room.

It is a test of whether you can use AI, teammates, tools, and time smartly.

The best hackathon teams are usually not chaotic geniuses. They are fast,
organized, and good at showing progress.

## Your Advantage

If you are strong at backend/product thinking, use that.

In an enterprise AI agent track, reliability matters more than flashy demos.

Your advantage:

- define the task clearly
- build a repeatable eval harness
- make measured improvements
- explain enterprise value
- avoid unnecessary complexity

## Day-One Strategy

### First 15 Minutes

Do not code immediately.

Ask:

- What is the exact input?
- What is the exact output?
- What counts as success?
- Is there hidden evaluation?
- Can we use external APIs?
- Are tools/retrieval allowed?
- What do judges care about most?

### First 45 Minutes

Build a baseline.

Even if it is bad, it gives you something to improve.

### First 2 Hours

Build the harness:

- load dataset
- run agent
- save outputs
- evaluate outputs
- list failures

### Middle Of Hackathon

Improve one failure type at a time.

Good sequence:

1. output schema
2. prompt rules
3. few-shot examples
4. retrieval/context
5. deterministic tools
6. verifier

### Final 90 Minutes

Stop experimenting.

Prepare:

- final run
- screenshots
- short demo
- score summary
- failure analysis
- enterprise value story

## What Judges Like

- clear problem understanding
- measured baseline
- measured improvement
- good error analysis
- reliability
- practical enterprise use case
- simple architecture that works
- honest limitations

## What Judges Usually Do Not Like

- vague claims
- no evals
- cherry-picked examples only
- overcomplicated multi-agent flow
- broken demo
- no enterprise relevance
- no explanation of reliability

## Personal Goal

Besides competing, meet people.

Talk to:

- teammates
- organizers
- mentors
- judges
- other builders

Ask what they are building. Tell them what you are building. Connect after.
