# Questions For Organizers

Ask these early. They prevent wasted work.

## API/Credits

- Are API credits provided to all participants, or only winners?
- Which providers are allowed?
- Are external paid APIs allowed?
- Are local models allowed?
- Is internet access allowed during final judging?

## Submission

- What exactly do we submit: code, notebook, hosted app, report, slides, or all?
- Do we need a public GitHub repo?
- Will judges run our code?
- If judges run code, what environment will they use?
- Do we need Docker?

## Dataset

- Is there a train/test split?
- Is there a hidden test set?
- Can we manually inspect all training rows?
- Can we create few-shot examples from the provided dataset?
- Are labels/ground truth provided?

## Evaluation

- What is the primary metric?
- Are qualitative judge scores also used?
- Are latency and cost considered?
- Are hallucinations penalized?
- Are safety/escalation behaviors rewarded?

## Agent Rules

- Are tools allowed?
- Is retrieval allowed?
- Is memory allowed?
- Are multiple model calls allowed per example?
- Is human-in-the-loop allowed?
- Are coding assistants like Claude Code, Codex, ChatGPT, or Cursor allowed?

## Demo

- How long is the final presentation?
- Should we show live demo or recorded demo?
- Should we show metrics?
- Should we explain architecture?
- Who are the judges: technical, business, investor, or mixed?
